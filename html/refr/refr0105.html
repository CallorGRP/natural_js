Alert(N.alert)

개요

Alert(이하 N.alert) 는 window.alert 이나 window.confirm과 같이 메세지를 다이얼로그로 표현하기 위한 컴포넌트 입니다.

API DEMO

생성자(Constructor)

Name Arguments Type Return Description
N.alert - - N.alert N.alert 객체를 생성합니다.
context window or JO - 기본옵션의 context 옵션과 같습니다.
msg (JO, HTML Tag String) or Object - 기본옵션의 msg 옵션과 같습니다.
vars Array Alert 기본옵션의 var 옵션과 같습니다.
N(obj).alert - - N.alert

N.alert을 플러그인 방식으로의 초기화 방식입니다.

N.alert과 동작되는 로직은 같으며 실제 N() 의 인자로 들어온 값이 N.alert()의 첫 번째 인자로 지정 됩니다.
msg (JO, HTML Tag String) or Object - N.alert의 msg 옵션과 같습니다.
vars Array - N.alert의 var 옵션과 같습니다.

기본옵션(Default options)

Name Type Default Required Description
context window or JO obj ○

Alert 의 배경이 되는 컨텐츠를 가려주는 overlay 가 적용될 부분을 jQuery selector 를 사용하여 지정 합니다.

window 객체를 지정하면 화면 전체를 덮는 Modal overlay 가 표시되고 JO를 입력하면 지정한 요소만 덮어줍니다.
msg (JO, HTML Tag String) or Object undefined ○

Alert 로 표시할 메시지

JO, HTML 문자열 이나 HTML 요소 모두 인식 합니다.

메시지의 변수를 vars로 지정한 값으로 치환 해줍니다.

N(window).alert({
	msg : "{0} 저는 {1}입니다.",
	vars : ["안녕하세요", "홍길동"]

/*
결과 메시지 : "안녕하세요 저는 홍길동 입니다."
*/
confirm boolean false true 지정 시 확인할 수 있도록 [확인], [취소] 버튼을 표시하고 false 지정 시 [취소] 버튼만 표시 합니다.
width int 0 메시지 다이얼로그의 넓이
height int 0 메시지 다이얼로그의 높이
title String undefined

메시지 다이얼로그의 제목

메시지 박스(context) 요소에 title 속성값이 정의 되어 있으면 정으된 값으로 지정되며 title 옵션이 undefined 이면 타이틀바를 표시하지 않습니다.
button boolean true 버튼 표시 유무
modal boolean true 모달 적용 유무
alwaysOnTop boolean false 메시지 다이얼로그를 항상 페이지의 최상위로 항상 유지하기 위한 옵션
alwaysOnTopCalcTarget String "div, span, ul, p" alwaysOnTop 옵션 적용 시 최상위 z-index를 계산하기 위한 대상 요소지정

jquery Selector 문자열로 지정
dynPos boolean true 브라우저 창 크기 조절 시 블록 오버레이의 크기도 같이 조절 할지 여부
draggable boolean false 메시지 창 드래그 가능 유무
onOk Function null 확인버튼을 눌렀을때 발생되는 이벤트
onCancel Function null 취소버튼을 눌렀을때 발생되는 이벤트

함수(Methods)

Name Arguments Type Return Description

hide - - N.alert 메시지 다이얼로그를 감춥니다.
remove - - N.alert 메시지 다이얼로그를 표현하는 HTML Element들을 제거합니다.

"ESC" 키를 눌러도 팝업이 remove 됩니다. N.alert 라이브러리를 공유하는 다른 컴포넌트(N.popup, N.grid(Validate), N.form(Validate))들도 이와 같은 동작을 합니다.

<a id="btnA">a</a>
<input id="btnInputButton" type="button" value="input button" />
<button id="btnButton">button</button>

DatePicker(N.datepicker)

개요

Datepicker(이하 N.datepicker)는 달력 다이얼로그를 입력요소 주변에 표시하여 마우스를 이용하여 날짜를 입력할 수 있게 해주는 컴포넌트입니다.

API DEMO

HTML 템플릿(HTML templete)

text input 요소에 Datepicker를 붙여 줍니다.

<input id="datepicker" type="text" />

N.datepicker - - N.datepicker N.datepicker 객체를 생성합니다.
context window or JO - 기본옵션의 context 옵션과 같습니다.
opts JSON Object - 기본 옵션을 JSON Object 형태로 지정 합니다.
N(obj).datepicker - - N.datepicker

N.datepicker을 플러그인 방식으로의 초기화 방식입니다.

N.datepicker와 동작되는 로직은 같으며 실제 N() 의 인자로 들어온 값이 N.datepicker()의 첫 번째 인자로 지정 됩니다.
opts JSON Object - N.datepicker의 opts 옵션과 같습니다.

focusin boolean true 날짜 입력요소에 자동으로 focusin 이벤트를 bind 하여 Datepicker를 표시할 것인지 여부를 지정 합니다.

}

Popup(N.popup)

개요

Popup(이하 N.popup) 은 지정한 요소나 다른페이지를 레이어팝업 형태로 만들어 주는 컴포넌트입니다.

url 옵션으로 지정한 페이지를 가져올때 가져온 페이지의 Controller 에는 caller(호출한 팝업의 instance(N.popup)) 와 opener(팝업을 호출한 페이지(부모페이지)의 Controller) 속성이 추가 되어 부모 페이지와 자신을 호출한 팝업 객체를 직접 제어할 수 있습니다.

opener 객체는 팝업을 생성하는 구문의 위치에 따라 생성이 안 될 수 있습니다.

API DEMO

생성자(Constructor)

Name Arguments Type Return Description
N.popup - - N.popup N.popup 객체를 생성합니다.
context window or JO - 기본옵션의 context 옵션과 같습니다.
opts JSON Object - 기본 옵션을 JSON Object 형태로 지정 합니다.
N(obj).popup - - N.popup

N.popup을 플러그인 방식으로의 초기화 방식입니다.

N.popup과 동작되는 로직은 같으며 실제 N() 의 인자로 들어온 값이 N.popup()의 첫 번째 인자로 지정 됩니다.
opts JSON Object - N.popup의 opts 옵션과 같습니다.

기본옵션(Default options)

Name Type Default Required Description
context JO obj 같은 페이지 내에 있는 블록(Block) 요소를 지정 합니다.

title String undefined

팝업의 제목,

title 옵션이 undefined 이면 타이틀바를 표시하지 않고 context 값으로 지정한 요소의 title 속성에 값이 있다면 해당 값으로 지정 됩니다.

button boolean true 버튼 표시 유무

dynPos boolean true 브라우저 창 크기 조절 시 블록 오버레이의 크기도 같이 조절 할지 여부
draggable boolean false 메시지 창 드래그 가능 유무
onOk Function null 확인버튼을 눌렀을때 발생되는 이벤트
onCancel Function null 취소버튼을 눌렀을때 발생되는 이벤트
onOpen String null

팝업이 열릴 때마다 발생되는 이벤트

이벤트 콜백 메서드는 팝업페이지의 Controller 에 구현되어야 합니다. 메서드면은 Controller 에 정의한 메서드명칭만 String 으로 지정 합니다.

이벤트 콜백의 메서드는 인자로 onOpenData 가 들어옵니다.

url을 지정하여 다른 페이지를 가져왔을때만 사용가능 합니다.
onOpenData Object null

onOpen 이벤트의 인자로 들어가 팝업에 전달될 데이터

url을 지정하여 다른 페이지를 가져왔을때만 사용가능 합니다.

open 함수의 첫번째 argument 로 넣어줘도 됩니다(동적인 데이터 전달 시 사용).
onClose Function null

팝업이 닫힐 때마다 발생되는 이벤트.

이벤트 콜백의 메서드는 인자로 onCloseData 가 들어옵니다.

팝업 페이지에서 자신을 닫을때는 Controller에서 다음과 같이 처리 합니다.

this.caller.close(onCloseData)

url을 지정하여 다른 페이지를 가져왔을때만 사용가능 합니다.
onCloseData Object null

onClose 이벤트의 인자로 들어가 팝업의 부모페이지에 전달될 데이터

url을 지정하여 다른 페이지를 가져왔을때만 사용가능 합니다.

close 함수의 첫번째 argument 로 넣어줘도 됩니다(동적인 데이터 전달 시 사용).

Name Arguments Type Return Description

Tab(N.tab)

개요

Tab(이하 N.tab) 지정한 HTML 탭 템플릿을 기반으로 탭을 만들어 주는 컴포넌트입니다.

API DEMO

HTML 템플릿(HTML templete)

다음과 같은 구조의 요소들을 배치하면 탭이 만들어 집니다.

<div id="tab">
	<ul>
		<li><a href="#tab1">Jeff Beck</a></li>
		<li data-opts='{ "onOpen" : "onOpen", "preload" : true, "url" : "html/apid/form.html" }'><a href="#tab2">Form</a></li>
		<li data-opts='{ "preload" : false, "url" : "html/apid/button.html", "*active" : true }'><a href="#tab3">Button</a></li>
	</ul>
	<div id="tab1"></div>
	<div id="tab2"></div>
	<div id="tab3"></div>
</div>

생성자(Constructor)

Name Arguments Type Return Description
N.tab - - N.tab N.tab 객체를 생성합니다.
context window or JO - 기본옵션의 context 옵션과 같습니다.
opts JSON Object - 기본 옵션을 JSON Object 형태로 지정 합니다.
N(obj).tab - - N.tab

N.tab을 플러그인 방식으로의 초기화 방식입니다.

N.tab과 동작되는 로직은 같으며 실제 N() 의 인자로 들어온 값이 N.tab()의 첫 번째 인자로 지정 됩니다.
opts JSON Object - N.tab의 opts 옵션과 같습니다.

기본옵션(Default options)

Name Type Default Required Description
context JO obj ○

같은 페이지 내에 있는 탭과 탭 컨텐츠를 감싸고 있는 "DIV" 요소를 지정 합니다.

탭플릿 요소는 다음과 같이 작성 합니다.

<div id="tab">
	<ul>
		<!-- 탭요소는  li 태그 로 표현 -->
		<li><a href="#tab1">Tab1</a></li>
		<li data-opts='{ "url" : "tab1.html" }'>
			<!-- 탭 컨텐츠와의 연결은 a TAG 로 표현
				href 속성에 해당 컨텐츠의 #id 입력 -->
			<a href="#tab2">Tab2</a>
		</li>
		<li data-opts='{ "url" : "tab2.html"}'>
			<a href="#tab3">Tab3</a>
		</li>
	</ul>
	<!-- 탭컨텐츠는  div 태그 로 표현 -->
	<div id="tab1">
		Tab1
	</div>
	<div id="tab2"></div>
	<div id="tab3"></div>
</div>
dataOpts Array []

선언형옵션을 명령형으로 지정할때 탭별로 탭의 순서대로 JSON Obejct 형태로 서브옵션을 지정 합니다.

N("#tab").tab({
	dataOpts : [
		{ width: "auto", url: "tab1.html", preload: false,
		  active: false, onOpen: undefined }, //Tab1,
		{ width: "auto", url: "tab2.html", preload: false,
		  active: false, onOpen: undefined }, //Tab2
		{ width: "auto", url: "tab3.html", preload: false,
		  active: false, onOpen: function() { } }  //Tab3
	]
});

선언형옵션은 탭요소(LI 태그)에 지정한다.
randomSel boolean false

true 이면 탭이 초기화 될 때 탭과 탭컨텐츠를 랜덤으로 보여줍니다. false 이면 첫 번째 탭을 보여줍니다.

탭에 선언형옵션으로 active 가 true 로 설정되어있으면 해당 옵션이 적용 됩니다.

effect boolean or Array false

탭 컨텐츠가 표시될때 전환 효과를 보여줄지 여부를 설정 합니다.

효과를 적용하지 않으려면 false를 입력하고 효과를 입력하려면 다음과 같이 입력합니다.

["이펙트", "시간(ms)", "이펙트가 끝날 때 마다 실행 될 콜백함수"]

N("#tab").tab({
	effect : ["slideDown", 300, null]
});

이펙트 종류 : show, hide, slideUp, slideDown, fadeIn, fadeOut
onActive Function null

탭이 활성화 될 때마다 실행되는 이벤트입니다.

이벤트 콜백 함수의 인자는 다음과 같이 반환 됩니다.

thisIdx, tab, content, tabs, content

arguments[0] : 활성화된 탭의 Index

arguments[1] : 활성화된 탭 요소

arguments[2] : 활성화된 탭 컨텐츠 요소

arguments[3] : 전체 탭 요소

arguments[4] : 전체 컨텐츠 요소

선언형옵션(Declarative options)

Name Type Default Description
url String undefined 특정 페이지를 탭컨텐츠로 만들경우 해당 페이지의 URL을 지정 합니다.

Pagination(N.pagination)

개요

Pagination(이하 N.pagination)은 데이터 표현에 대한 브라우저의 부하를 줄이기 위해 데이터를 지정한 갯수만큼 잘라서 보여주는 컴포넌트입니다. N.grid 와 연동하여 사용하면 서버단 페이징(DB 쿼리 페이징) 및 브라우저 단 페이징(전체 조회 후 잘라서 보여주기) 을 쉽게 구현 할 수 있습니다. 또한 페이징이 필요한 어떠한 목록도 페이징 처리를 할 수 있습니다.

		<li><a href="#" title="이전">prev</a></li>
	</ul>
	<ul>
		<li><a href="#"><span>1</span></a></li>
	</ul>
	<ul>
		<li><a href="#" title="다음페이지">next</a></li>
		<li><a href="#" title="마지막페이지">last</a></li>
	</ul>
</div>
